## 2026-04-27 - Global Error Handler missing defensive behavior
**Vulnerability:** The application lacks a global error handler that safely catches synchronous errors and prevents the leakage of stack traces.
**Learning:** In memory, it states: `The Express server utilizes a custom global error handler ('globalErrorHandler' in 'server/_core/errorHandler.ts') applied at the end of the middleware chain in 'server/_core/index.ts'. To prevent leaking stack traces while avoiding application crashes, it defensively delegates to 'next(err)' if 'res.headersSent' is true, and otherwise returns a generic 500 JSON error ('{ error: "Internal Server Error" }').` but `server/_core/errorHandler.ts` was not found.
**Prevention:** Always verify the presence and proper application of the global error handler. If it's missing, add it to prevent stack traces from leaking and to fail securely.
