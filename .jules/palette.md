## 2024-04-21 - Accessible Dropdown Trigger Context and Generic Component Focus Indicators
**Learning:** In a collapsible sidebar pattern, dropdown triggers representing user menus often rely heavily on adjacent text (e.g., username/email) for context, which gets hidden via CSS or conditionally unrendered when the sidebar is minimized. Furthermore, dynamically rendered fallback/error boundary elements often overlook default application focus rings when generic HTML `<button>` tags are used instead of structured library components (e.g., Shadcn/ui).
**Action:** When working with collapsible sidebars or responsive UI sections where text is visually hidden based on states (like `group-data-[collapsible=icon]:hidden`), ensure the outer interactive element always has a permanent `aria-label` providing full context for screen readers. Additionally, always explicitly add `focus-visible:ring` and related outline reset utilities to any generic `<button>` tag that isn't a pre-configured library component, particularly in global boundaries like `ErrorBoundary.tsx`.

## 2026-04-20 - Added missing ARIA attributes and focus styles to AIChatBox
**Learning:** The default AI chat box implementation in AIChatBox lacks basic semantic A11y support for its Textarea input and generic Send icon button, leading to poor screen reader output and limited discoverability. Furthermore, the suggested prompts buttons lacked focus visibility for keyboard navigation.
**Action:** Add explicit `aria-label` to text areas and icon-only buttons like 'Send'. Ensure all interactive elements, like dynamically mapped generic `<button>` tags, receive `focus-visible:ring` classes for keyboard accessibility.
